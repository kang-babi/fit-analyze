# VALIDATION.md

# Analysis Validation Specification

Version: 1.0.0

---

# Purpose

This document defines the validation process used to evaluate the
quality, completeness, and consistency of generated activity analyses.

The purpose of validation is not to determine whether an athlete
performed well.

Instead, validation ensures that every analysis adheres to the project's
methodology, coaching philosophy, and reporting standards.

Validation should be performed after every activity analysis and before
the final report is considered complete.

---

# Validation Principles

Every report should be:

complete

consistent

evidence-based

objective

reproducible

transparent

Validation should never depend on subjective opinion.

---

# Validation Workflow

```
Activity Analysis

↓

Validate Structure

↓

Validate Evidence

↓

Validate Interpretation

↓

Validate Recommendations

↓

Validate Consistency

↓

Assign Validation Status

↓

Publish Report
```

Every stage should succeed before the report is finalized.

Structure

3.

Each category contributes equally to the overall validation result.

---

# Structure Validation

Purpose

Ensure that every report follows REPORT.md.

Required Sections

✓ Activity Overview

✓ Workout Summary

✓ Workout Execution

✓ Physiological Response

✓ Running Mechanics

✓ Pacing Analysis

✓ Historical Comparison

✓ Long-Term Trend

✓ Recommendations

✓ Overall Assessment

Missing sections should be reported.

---

# Evidence Validation

Purpose

Ensure that every conclusion is supported by measurable observations.

Examples

Good

Cadence decreased by 7 spm during the final interval, suggesting
fatigue.

Poor

Running form collapsed.

Every interpretation should reference objective evidence whenever
possible.

---

# Interpretation Validation

Purpose

Verify that interpretations are reasonable.

Interpretations should:

follow observations

acknowledge uncertainty

avoid unsupported assumptions

consider historical context

Avoid

Heart rate increased because hydration was poor.

Prefer

Heart rate increased. Possible contributing factors include heat,
fatigue, hydration, or pacing.

---

# Recommendation Validation

Purpose

Ensure recommendations follow COACHING.md.

Recommendations should:

be actionable

be conservative

respect the current training plan

be supported by evidence

avoid unnecessary changes

Reject recommendations that contradict higher coaching priorities.

---

# Consistency Validation

Purpose

Verify internal consistency.

Examples

A report should not simultaneously state:

The workout was well paced.

and

Pacing was highly inconsistent.

Contradictions should be resolved before publication.

---

# Confidence Validation

Purpose

Ensure confidence reflects available evidence.

High

Complete observations available.

Medium

Minor observations missing.

Low

Important observations unavailable.

Unknown

Evidence insufficient.

Confidence should never exceed the quality of available data.

---

# Missing Data

Reports should explicitly identify important missing observations.

Examples

Ground Contact Time unavailable.

Temperature unavailable.

Workout definition unavailable.

Missing observations should reduce confidence rather than invalidate the
analysis.

---

# Trend Validation

Trend analysis requires historical evidence.

Do not infer trends from a single activity.

Verify that comparisons involve equivalent workout types whenever
possible.

Examples

Easy Run

↓

Easy Run

Threshold

↓

Threshold

Avoid comparing unrelated workout objectives.

---

# Coaching Validation

Verify that recommendations follow coaching philosophy.

Questions

Does the recommendation prioritize consistency?

Does it minimize injury risk?

Does it respect the current training plan?

Does it avoid compensating for missed workouts?

Does it encourage gradual progression?

Every answer should be affirmative.

---

# Report Completeness

A report is complete when it answers the following questions.

What happened?

Why did it happen?

How does it compare historically?

Does it matter?

What should happen next?

If any question remains unanswered, the report is incomplete.

## Interpretation

## Recommendations

✓ Actionable

✓ Evidence-based

---

---

# Validation Status

Reports should receive one of the following statuses.

Pass

All validation criteria satisfied.

Pass with Warnings

Minor issues identified.

Review Required

Significant inconsistencies detected.

Incomplete

Critical sections or evidence missing.

---

# Common Validation Failures

Examples include:

Making recommendations without evidence.

Ignoring historical context.

Confusing observations with interpretations.

Drawing conclusions from incomplete data.

Contradicting coaching philosophy.

Comparing unrelated workout types.

Failing to acknowledge uncertainty.

---

# Quality Goals

Every finalized report should be:

accurate

repeatable

internally consistent

supported by evidence

aligned with project methodology

understandable to the athlete

---

# Success Criteria

A validated report should allow another reviewer to independently reach
the same conclusions using the same activity data.

The analysis should be explainable, reproducible, and defensible.

---

# Guiding Principle

Validation exists to ensure confidence in the analysis rather than
confidence in the athlete.

A report should be trusted because its reasoning is transparent,
evidence-based, and internally consistent.
