# Running Activity Report Specification

Version: 1.0.0

---

# Purpose

This document defines the standard report format generated after
analyzing a running activity.

The report should be:

- consistent
- objective
- reproducible
- coaching-oriented
- comparable across activities

The report format is independent of the activity type.

---

# Report Philosophy

A report is not a collection of statistics.

It is an explanation of:

- what happened
- why it happened
- whether it was expected
- what should happen next

Every section should progress from objective observations to coaching
recommendations.

---

# Report Structure

Every activity report should contain the following sections.

1. Activity Overview

2. Workout Summary

3. Workout Execution

4. Physiological Response

5. Running Mechanics

6. Pacing Analysis

7. Environmental Context

8. Strengths

9. Opportunities for Improvement

10. Historical Comparison

11. Long-Term Trend

12. Recommendations

13. Overall Assessment

---

# Activity Overview

Purpose

Provide a concise summary of the activity.

Contents

- Date
- Activity Type
- Duration
- Distance
- Average Pace
- Average HR
- Average Cadence
- Average Power
- Elevation Gain

Example

The activity covered 8.2 km in 58:43 with an average pace of
7:09/km.

Average heart rate was 149 bpm with a cadence of 166 spm.

---

# Strengths

Purpose

Highlight positive observations.

Examples

Excellent pacing.

Consistent cadence.

Minimal HR drift.

Strong finish.

Workout completed exactly as prescribed.

Strengths should always be evidence-based.

---

# Opportunities for Improvement

Purpose

Identify areas that could improve future performance.

Examples

Recovery pace too fast.

Cadence dropped during the final interval.

Heart rate rose earlier than expected.

Started the workout too aggressively.

Avoid speculative conclusions.

---

# Recommendations

Purpose

Provide actionable coaching advice.

Recommendations should:

prioritize consistency

avoid drastic changes

consider training phase

consider accumulated fatigue

Examples

Maintain current easy pace.

Continue progressing long runs.

Focus on recovery between threshold intervals.

Increase cadence awareness during fatigued segments.

Recommendations should never contradict the current training plan
without strong evidence.

---

# Overall Assessment

Purpose

Summarize the activity.

Include

Workout Rating

Execution Rating

Confidence

Primary Takeaway

Example

This was a well-executed threshold session.

Although fatigue appeared during the final interval, pacing remained
consistent and the workout objective was achieved.

Overall execution was excellent.

---

# Confidence

Every major conclusion should include a confidence level.

High

Complete data available.

Medium

Minor missing observations.

Low

Important measurements unavailable.

Unknown

Insufficient evidence.

Confidence applies to conclusions, not activities.

---

# Writing Guidelines

Reports should:

be objective

be concise

avoid unnecessary jargon

support every conclusion with evidence

clearly distinguish facts from interpretation

avoid emotional language

avoid exaggerated praise or criticism

---

# Report Order

Every report should follow this sequence.

Observation

↓

Evidence

↓

Interpretation

↓

Recommendation

Never reverse this order.

# Guiding Principle

A report should answer three questions.

What happened?

Why did it happen?

What should the athlete do next?

If these questions cannot be answered, the report is incomplete.
